What to do if you have a problem

If you encounter a problem with anything to do with AssayReg, and this help document does not provide the answer you need, please contact ChemBio Hub by email at info@chembiohub.ox.ac.uk

Logging in and initial screen

When you first successfully log onto AssayReg, you will see a list of available projects. A system administrator is responsible for adding you to projects, which will then be displayed in this list. If you think you should be able to access a project via this list and can't, please contact ChemBio Hub.

Each project has a number of options available to you:

  • Project Overview - click the project title to be taken to the Project Overview page;
  • Add - Add data to the project via the Project Overview page, subject to the correct access permisions;
  • Search - Search data in the project via the search page - the project will be preselected, enabling you to filter results further.

Project Data Overview

For each project, there is a data overview page. This displays all of the data present in the project, separated out into three levels: Study, Assay and Activity.

Project sections - an explanation

Each Project contains 3 sections:

  • Study - A study normally holds details for a collection of different assays, for example a set of different Characterisation experiments on a particular drug target.
  • Assay - An assay normally holds details of experimental protocols and method development, for example data which needs to be provided to make the assay reproducible. Assays can belong to multiple different studies.
  • Activity - An activity holds the outcome of one pass of an assay, usually machine readouts or more qualitative data. Activity data is specific to a given Assay.

Show details

You can see the data for a particular level by clicking the appropriate Actions button and clicking View Details. This will open a popup window with the data loaded.

Activity

Activity data for a particular assay is displayed as a table. You can page through the data easily using the buttons at the bottom of the table. If your table contains a lot of data, you can use the scrollbar at the bottom of the table to slide the table from side to side to see all the data.

Each row of the Activities table contains a Duplicate link in the first column. Clicking this allows you to add a new Activity data point via the Add Single Activity form, with the data from this row prepopulated into the form. Adding Data is discussed in more depth below.

If your Activity data contains a UOX ID as used in ChemReg, the system will attempt to load a structure image of that identifier into the table.

Filtering activities

At the top of the Activities section, there is a text box which allows you to filter the Activities based on text entered here. So for example, if you type "dichloro" into the text box, the Activities table will only show rows that contain "dichloro" in any of the columns.

Export

Next to the text box on the activities table, there is a Download button. This allows you to download all the results in the table as an Excel file. The downloaded file will include any text filtering you have added - so if you have typed "dichloro" into the search box, your downloaded results will contain only results with "dichloro" in any of the fields. To download unfiltered results, just delete any text from the text box.

Adding data

If you have the correct permissions, you can add data to the system through the AssayReg user interface.

Single record

Adding a single record is the simplest means of getting data into the system. Click Add Single Activity and select the correct Assay template to use from the list shown.

A form is then displayed with blank fields, each field has a brief explanation of what it represents. Specific datatypes may have special inputs - dates for example may show a datepicker/calendar field.

When you have added the necessary data, click the Submit button. Your data will be saved and you will be able to view it in the data overview page.

Multiple records (via upload)

To add multiple records to AssayReg in one go, you will need your data in an Excel format. You are able to download a template of the data format by clicking Upload Multiple Activities then click Download (activity) Template if you would like all of the fields which have been set up for the Assay listed. You can then use this to add your data for a smooth upload.

If instead you have data within another spreadsheet, for example from another lab or from machine outputs, with extra fields that are not required for the Assay you're adding data to, you can also upload that data but you may need to map your data, which is explained later in this document.

For simple file uploads, follow these steps:

  • Select the Sub-project and Assay you wish to register the data to. Select ‘Upload multiple activities’. Select the Assay definition you wish to submit your data to.

  • Select the Assay definition you wish to submit your data to.
  • A file upload dialog should now be shown. Click upload and choose your data file.

  • If your file contains multiple tabs, select the appropriate one. The data will be previewed below.

  • If there are no unmapped required fields for this upload, you can click ‘Save this sheet’. Your data is now registered to your project in AssayReg.

Data mapping

If you are uploading data for your Activities from a file containing lots of supplementary data, you may have to specify which columns in your file correspond to which Activity columns. This is especially important where there are mandatory fields for the specified Activity - that is to say, fields which cannot be empty for a single result.

AssayReg uses data mapping to help with data integrity and also to allow conversion to formats required by public databases such as PubChem and ChEMBL.

Auto mapping

AssayReg will attempt to "automap" your columns to the Activity columns. Columns in your file which match columns in the Activity correctly have a green mapping icon . Columns in your file which cannot be automapped have a yellow mapping icon .

You may want to map a column in your file to an Activity column manually if the names are slightly different - for example, you may have a column called "1st Run" in your file and a column called "1st Observation" in the Activity table. They have the same meaning for your work but have been labelled differently. To do this, click the mapping icon in the column you want to map.

Depositing data to ChEMBL

  • To view data in a ChEMBL format select the search ‘Go’ icon.
  • You are now in the AssayReg serach page. Select ‘View as ChEMBL data’
  • Your activity data should now be visible as a JSON schema form. This is the format in which it will be deposited to ChEMBL.
